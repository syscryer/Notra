# Notra Editor UI Override

## 产品类型

桌面文本编辑器与搜索替换工作台。核心用户是开发者、运维、数据处理人员和经常处理日志/配置/脚本的高级用户。

## 覆盖 Master 的规则

- 布局优先级高于装饰：第一屏必须直接呈现菜单工具栏、标签栏、编辑器和状态栏。
- 浅色模式优先，深色模式作为可选主题；不能把深色作为默认主视觉。
- 不使用作品集、营销页、hero、项目卡片、悬浮大卡片模式。
- 信息密度接近 mxterm/Notepad++，但分区、间距和状态反馈保持现代工具质感。
- 主窗口默认像 Notepad++ 一样干净；未打开目录时不显示文件树、右侧工具轨、设置面板或搜索输入区。
- 打开目录后允许出现 VS Code 风格左侧活动栏和文件树，文件树必须可一键关闭并回到无目录编辑模式。
- 主编辑区是铺满剩余空间的编辑画布，不使用默认输入框边框、卡片边框或悬浮容器。
- 搜索替换是一级能力：入口要显眼，但查找/替换输入必须在 Notepad++ 风格浮动工具窗内，不占用主编辑界面的常驻区域。
- 语言语法高亮是一级能力：工具栏或菜单必须能直接切换当前文档语言，并支持 None 普通文本、常用语言、按字母分组和用户自定义语言。
- 编码是一级能力：必须区分“使用某编码重新解释当前文件”和“转为某编码并改写文件”，不能把两类动作混在一起。
- 查找结果、替换预览和日志只在触发搜索/替换后从底部按需出现。
- 批量操作必须有预览、后台执行、跳过原因和失败日志。
- 只读、大文件保护、后台任务、未保存退出等状态必须在界面上明确可见。

## 交互检查

- 高频按钮保留中文文本和 tooltip。
- 工具栏提供打开目录/关闭目录入口；未打开目录时编辑器不能留下空白侧栏。
- 工具栏或状态栏提供语言和编码入口；状态栏同步显示当前语言、编码、换行符和编辑状态。
- 编码菜单必须包含 ANSI、UTF-8、UTF-8-BOM、UTF-16 Big Endian、UTF-16 Little Endian，并保留编码字符集入口。
- `Ctrl+F`、`Ctrl+H`、`Ctrl+Shift+F` 打开浮动查找/替换工具窗。
- 关闭 tab 与关闭应用都必须保护未保存内容。
- 目录搜索、目录替换预览、目录替换写回不能阻塞编辑器。
- 只读文档允许查找和复制，但编辑、保存、替换入口要明确禁用或提示。
- 结果列表点击后必须切换到对应文件并选中当前命中。
