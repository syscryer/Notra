# 产品原则

## 定位

Notra 是一个 Rust 原生文本编辑器，目标是做到漂亮、便捷、速度快。

它不是复刻 Notepad++ 的全部历史包袱，而是把 Notepad++ 最顺手的搜索与替换工作流作为核心能力，再用现代工具型 UI 和更可控的 Rust 架构重新实现。

## 三条原则

1. 速度快是底线：启动、打开文件、输入、滚动、搜索和替换都不能让用户等。
2. 使用便捷是核心：高频操作必须直接、稳定、少打断，搜索替换要作为一等功能设计。
3. 外观漂亮是第一印象：整体风格参考 Catio 和 mxterm，干净、轻量、现代，但不牺牲编辑区密度。

## v0.1 验收范围

- 多 Tab 编辑。
- 新建、打开、保存、另存为。
- 当前文件查找、查找上一个、查找下一个、全部查找。
- 当前文件替换、替换当前、全部替换。
- 目录查找。
- 目录批量替换预览，确认后执行。
- 搜索结果底部面板，按文件和行展示。
- 行号、状态栏、编码、换行符、文件大小。
- 浅色现代 UI，支持暗色主题切换。

## 暂不做

- 插件系统。
- 宏。
- FTP。
- 复杂 AI 功能。
- 复杂设置中心。

## 体验基准

- 查找替换入口固定：`Ctrl+F`、`Ctrl+H`、`Ctrl+Shift+F`、`Ctrl+Shift+H`。
- 支持普通、扩展、正则三种搜索模式。
- 支持大小写、全词匹配、循环搜索、目录过滤。
- 批量替换必须先预览，避免误操作。
- 任何批量操作都要有明确统计和状态反馈。
