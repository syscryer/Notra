# 04 搜索替换

## Notepad++ 行为参考

参考 notepad-- 源码中的 `findwin`、`dirfindfile`、`findresultwin`、`batchfindreplace`，只提炼行为，不复制实现。

Notra v0.1 落地：

- 当前文件查找。
- 当前文件替换。
- 当前文件全部替换。
- 打开文档查找。
- 打开文档全部替换。
- 目录查找。
- 目录替换预览。
- 目录替换确认执行。
- Notepad++ 风格浮动查找/替换窗口：查找、替换、文件查找、工程中查找、标记。
- 普通、扩展、正则三种模式。
- 大小写和全词匹配。
- 文件过滤、跳过目录、是否递归子目录、最大文件大小限制。
- 目录搜索会跳过疑似二进制文件。
- 目录替换预览会跳过只读文件。
- 只读或大文件保护状态下，当前文件替换入口会禁用并提示原因。
- 结果面板按文件、行、列展示。
- 当前命中会在编辑器中高亮并选中，结果列表点击会打开文件并定位命中。
- 打开文档结果按 tab 分组，点击命中会切换到对应文档。
- 目录搜索、目录替换预览、目录替换写回使用后台任务，避免阻塞编辑器输入和滚动。
- 历史面板展示查找历史和替换历史，点击即可回填到浮动查找窗口。
- 设置面板可以调整默认搜索选项、目录过滤参数和主题。

## 扩展搜索语法

- `\n` 换行。
- `\r` 回车。
- `\t` 制表符。
- `\0` 空字符。
- `\\` 反斜杠。

扩展模式下，替换内容同样支持以上转义。

## 正则替换

- 支持 Rust regex 语法。
- 替换内容支持 `$1`、`$name` 等捕获组。
- 正则错误直接展示给用户，不静默降级。

## 替换安全策略

- 当前文件替换写入 undo 栈。
- 目录替换必须先预览。
- 目录替换失败的文件进入日志。
- 目录替换写回采用临时文件、原文件备份、替换失败回滚流程。
- 疑似二进制文件不参与目录搜索和替换。
- 只读文件不进入目录替换预览。
- 空查询不执行。
- 目录批量写回完成后，会重新加载已打开的受影响文件。
