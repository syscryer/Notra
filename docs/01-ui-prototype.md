# 01 UI 原型

## 设计方向

Notra 使用 Notepad++ 风格的干净主窗口，叠加 Catio 的现代质感和 mxterm 的高密度工具审美。
UI/UX Pro Max 设计系统已落到 `design-system/notra/MASTER.md`，Notra 编辑器页的实际约束以 `design-system/notra/pages/editor.md` 为准。

## 交互规则

- 所有高频按钮必须有文本，不只放图标。
- 小图标按钮必须有 tooltip。
- 主窗口默认保持 Notepad++ 式干净结构，不常驻文件树、右侧工具轨或设置面板。
- 编辑区必须铺满标签栏和状态栏之间的剩余空间，不呈现为带默认边框的输入框。
- 搜索替换使用 Notepad++ 风格的浮动工具窗，不常驻主界面底部。
- 批量替换先生成预览，用户确认后执行。
- 结果面板使用单击选择、双击打开的桌面工具习惯。
- 后台目录任务运行时保留编辑器可操作，并在结果/日志面板显示任务状态。
- 搜索历史和替换历史可在浮动查找窗口内回填，不藏进设置。
- 打开目录只作为目录搜索上下文，不在主窗口展开文件树。
- 只读、大文件保护、后台任务、未保存退出必须有明确状态提示。
- 关闭应用时如果存在未保存文档，必须提供全部保存并退出、不保存退出、取消。
- 动效克制，只用于 hover、选中、面板切换。

## UI/UX 检查

- 对比度满足普通文本 4.5:1。
- 控件状态明确：默认、hover、active、disabled。
- 主编辑区不被说明文字占据。
- 查找输入框不占用主界面常驻区域。
- 视觉不使用 emoji 作为结构图标。
- 页面不使用装饰性渐变球或营销式大 hero。
