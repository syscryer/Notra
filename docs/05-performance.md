# 05 性能验证

## 目标

- 应用启动：优先保持轻量。
- 当前文件查找：普通文本即时反馈。
- 目录搜索：流式生成结果，不阻塞主流程。
- 批量替换：先预览，再写回，避免误操作。

## 当前实现策略

- 搜索核心在 `notra-core`，方便单独测试和优化。
- 目录扫描限制单文件大小，默认跳过 20MB 以上文件。
- 搜索结果只保存必要上下文。
- 目录搜索、目录替换预览、目录替换写回在 app 层使用后台线程执行。
- UI 与 core 分层，后续可以替换编辑区渲染实现。
- 超过 20MB 的打开文件默认进入只读保护，避免当前 egui 文本编辑控件在大文件上误触发重写成本。
- 批量写回采用临时文件与备份替换流程，性能优先级低于数据安全。

## 验证命令

```powershell
cargo test --workspace
cargo check --workspace
cargo run -p notra-core --example perf_smoke
```

`perf_smoke` 会生成约 12 万行内存文本和 160 个临时文本文件，验证内存查找、内存替换和目录搜索耗时。

## 当前烟测结果

Windows 本机 debug 构建下最近一次结果：

- 12 万命中内存查找：约 458ms。
- 12 万命中内存替换：约 452ms。
- 160 个文件目录搜索：约 110ms。

核心搜索已改为一次构建行索引，避免每个命中重复扫描全文计算行号。
